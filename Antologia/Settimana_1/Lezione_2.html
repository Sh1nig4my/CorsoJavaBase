Lezione 2: Utilizzo di Git e Risoluzione Problemi

1. Introduzione a Git

Git è uno strumento di controllo di versione distribuito progettato per fornire velocità, efficienza e flessibilità nello sviluppo software. Consente agli sviluppatori di tracciare ogni modifica apportata ai file di un progetto, di tornare a versioni precedenti del codice e di collaborare in modo efficace con altri membri del team.

Esempio:

Clonare un repository GitHub e apportare modifiche:

# Clonare un repository
$ git clone https://github.com/user/repository.git

# Creare un nuovo file
$ echo "Hello, Git!" > hello.txt

# Aggiungere il file al controllo di versione
$ git add hello.txt

# Fare un commit
$ git commit -m "Aggiunto il file hello.txt"

# Caricare le modifiche su GitHub
$ git push origin main

Esempio Avanzato:

Gestire una situazione di conflitto:

# Modificare un file in due branch diversi
$ git merge branch_conflitto

# Messaggio di conflitto
# Risolvere il conflitto modificando il file manualmente

# Aggiungere le modifiche risolte
$ git add file_conflitto.txt

# Completare il merge
$ git commit -m "Conflitto risolto"

2. Repository GitHub

GitHub è una piattaforma online che ospita repository Git, offrendo strumenti avanzati per la collaborazione e la gestione di progetti software.

Esempio:

Creazione di un nuovo repository su GitHub:

  1. Accedi a GitHub e fai clic su "New Repository".
  2. Inserisci un nome per il progetto e scegli la visibilità.
  3. Seleziona opzioni aggiuntive, come l'aggiunta di un file README.
  4. Clicca su "Create Repository" per iniziare.

Esempio Avanzato:

Abilitare GitHub Actions per un workflow automatizzato:

# Creare un file di configurazione workflow
.github/workflows/build.yml:

name: Build
on:
  push:
    branches:
      - main
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2
      - name: Setup Java
        uses: actions/setup-java@v2
        with:
          java-version: '11'
      - name: Build project
        run: mvn clean install

Esercizio Pratico

Configura un repository Git seguendo questi passaggi:

  1. Inizializza un nuovo repository Git.
  2. Crea un file README.md e aggiungi una breve descrizione del progetto.
  3. Fai un commit e carica il repository su GitHub.

3. Risoluzione Problemi con il Pseudocodice

Il pseudocodice è una rappresentazione semplificata di un algoritmo che utilizza un linguaggio naturale e strutture logiche per descrivere la soluzione a un problema.

Esempio:

Pseudocodice per calcolare la somma di un array:

1. Inizializza somma a 0
2. Per ogni numero nell'array:
   a. Aggiungi il numero alla somma
3. Stampa il valore della somma

Esempio Avanzato:

Pseudocodice per trovare il massimo in un array:

1. Inizializza massimo al primo elemento dell'array
2. Per ogni elemento nell'array:
   a. Se l'elemento è maggiore di massimo, aggiorna massimo
3. Stampa il valore massimo
