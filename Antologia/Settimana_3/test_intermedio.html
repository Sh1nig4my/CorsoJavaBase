Test Intermedio - Corso di Programmazione Backend

Descrizione del Test

Il test intermedio ha l'obiettivo di valutare le conoscenze acquisite nelle lezioni precedenti riguardo i concetti fondamentali di Java e JDBC. Il test è suddiviso in due parti:

  • Parte 1: Domande teoriche per verificare la comprensione dei concetti fondamentali.
  • Parte 2: Esercizi pratici di programmazione per testare l'applicazione delle conoscenze apprese.

Tempo a disposizione: 2 ore e 30 minuti.

Parte 1: Domande Teoriche (30 minuti)

Rispondi alle seguenti domande in modo chiaro e dettagliato.

  1. Spiega la differenza tra ArrayList e HashMap in Java. Quali sono i vantaggi di ciascuna?
  2. Descrivi il concetto di Programmazione Orientata agli Oggetti e i suoi quattro principi fondamentali.
  3. Qual è il ruolo di JDBC nella comunicazione tra un'applicazione Java e un database? Fornisci un esempio pratico di connessione a MySQL.
  4. Cos'è un Optional in Java e come può essere utilizzato per gestire valori nulli in modo sicuro?

Parte 2: Esercizi Pratici (2 ore)

Scrivi i seguenti programmi in Java.

Esercizio 1: Gestione di una Lista di Studenti

Scrivi una classe Studente con i seguenti attributi:

  • id (intero, generato automaticamente)
  • nome (String)
  • cognome (String)
  • email (String, univoco)

Implementa una classe GestioneStudenti che permetta di:

  • Aggiungere un nuovo studente
  • Elencare tutti gli studenti
  • Ricercare uno studente per email

Esercizio 2: Connessione a un Database con JDBC

Scrivi un programma che:

  • Si connetta a un database MySQL chiamato scuola
  • Crei una tabella studenti se non esiste
  • Inserisca alcuni studenti nella tabella
  • Visualizzi tutti gli studenti presenti nel database

Utilizza le best practices per la gestione delle connessioni e delle eccezioni.

Consigli Utili e Esempi Pratici

Ecco alcuni consigli per affrontare al meglio il test:

  • Leggi attentamente ogni domanda prima di rispondere.
  • Suddividi il problema in sottoproblemi più piccoli per una soluzione più chiara.
  • Scrivi codice ben strutturato e commentato per facilitarne la comprensione.
  • Utilizza le collezioni e le best practices per ottimizzare la gestione dei dati.
  • Testa i tuoi programmi con diversi casi di input per verificarne la correttezza.

Esempio: Connessione a un Database MySQL con JDBC


    import java.sql.*;
    
    public class ConnessioneDatabase {
        private static final String URL = "jdbc:mysql://localhost:3306/scuola";
        private static final String USER = "root";
        private static final String PASSWORD = "password";
    
        public static void main(String[] args) {
            try (Connection conn = DriverManager.getConnection(URL, USER, PASSWORD)) {
                System.out.println("Connessione avvenuta con successo!");
    
                String query = "SELECT * FROM studenti";
                try (Statement stmt = conn.createStatement(); ResultSet rs = stmt.executeQuery(query)) {
                    while (rs.next()) {
                        System.out.println(rs.getInt("id") + " - " + rs.getString("nome") + " " + rs.getString("cognome"));
                    }
                }
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }

Esempio: Gestione delle Eccezioni


    public class GestioneEccezioni {
        public static void main(String[] args) {
            try {
                int risultato = dividi(10, 0);
                System.out.println("Risultato: " + risultato);
            } catch (ArithmeticException e) {
                System.out.println("Errore: divisione per zero non consentita.");
            }
        }
    
        public static int dividi(int a, int b) throws ArithmeticException {
            if (b == 0) {
                throw new ArithmeticException("Impossibile dividere per zero.");
            }
            return a / b;
        }
    }
