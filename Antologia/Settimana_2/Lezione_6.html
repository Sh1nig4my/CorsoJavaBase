1. Fondamenti di Java

1.1 Classi, Metodi e Costruttori

Le classi in Java sono i pilastri fondamentali della programmazione orientata agli oggetti. Rappresentano un modello o uno schema per creare oggetti e incapsulano attributi e comportamenti comuni. I metodi sono blocchi di codice definiti all'interno delle classi per eseguire azioni specifiche, mentre i costruttori sono metodi speciali utilizzati per inizializzare nuovi oggetti della classe. Un costruttore viene chiamato automaticamente al momento della creazione dell'oggetto e può essere sovraccaricato per gestire diversi scenari di inizializzazione.

public class Veicolo {
            private String modello;
            private int anno;
        
            // Costruttore
            public Veicolo(String modello, int anno) {
                this.modello = modello;
                this.anno = anno;
            }
        
            // Metodo
            public void descriviVeicolo() {
                System.out.println("Modello: " + modello + ", Anno: " + anno);
            }
        }
        
        public class Main {
            public static void main(String[] args) {
                Veicolo veicolo = new Veicolo("Toyota", 2021);
                veicolo.descriviVeicolo();
            }
        }

1.2 Interfacce

Le interfacce in Java rappresentano un contratto che una classe può implementare. Definiscono un insieme di metodi che devono essere implementati dalle classi che le utilizzano. Le interfacce sono essenziali per raggiungere il polimorfismo e favorire una progettazione modulare e flessibile. Inoltre, permettono la creazione di comportamenti comuni tra classi non correlate, favorendo la riusabilità del codice.

public interface Strumento {
            void suona();
        }
        
        public class Chitarra implements Strumento {
            @Override
            public void suona() {
                System.out.println("Strimpella una melodia.");
            }
        }
        
        public class Main {
            public static void main(String[] args) {
                Strumento strumento = new Chitarra();
                strumento.suona();
            }
        }

1.3 Classi Astratte

Le classi astratte, a differenza delle interfacce, possono contenere metodi con implementazioni concrete oltre a metodi astratti. Questo consente di fornire una struttura di base comune a tutte le sottoclassi, riducendo la duplicazione del codice. Le classi astratte non possono essere istanziate direttamente, ma devono essere estese da altre classi concrete che implementano i metodi astratti definiti.

public abstract class Animale {
            abstract void verso();
        
            public void dormi() {
                System.out.println("Zzz...");
            }
        }
        
        public class Gatto extends Animale {
            @Override
            void verso() {
                System.out.println("Miao");
            }
        }
        
        public class Main {
            public static void main(String[] args) {
                Animale gatto = new Gatto();
                gatto.verso();
                gatto.dormi();
            }
        }

2. Introduzione alle Collezioni

Le Collezioni in Java sono parte integrante del framework java.util. Offrono strutture dati flessibili e potenti per memorizzare, manipolare e recuperare gruppi di oggetti. Le collezioni sono progettate per ottimizzare le prestazioni, ridurre gli errori e semplificare la gestione dei dati. In questa sezione, esploreremo in dettaglio il framework delle collezioni, con particolare attenzione a ArrayList, HashMap e HashSet.

Framework delle Collezioni

Il framework delle collezioni fornisce diverse interfacce e implementazioni, tra cui:

  • List: Una collezione ordinata che consente duplicati. Esempi: ArrayList, LinkedList.
  • Set: Una collezione non ordinata che non consente duplicati. Esempi: HashSet, TreeSet.
  • Map: Una struttura dati che associa chiavi a valori. Esempi: HashMap, TreeMap.

Esempi Pratici

Utilizzo di ArrayList


        import java.util.ArrayList;
        
        public class ArrayListExample {
            public static void main(String[] args) {
                ArrayList fruits = new ArrayList<>();
                fruits.add("Mela");
                fruits.add("Banana");
                fruits.add("Arancia");
        
                System.out.println("Lista della frutta: " + fruits);
                fruits.remove("Banana");
                System.out.println("Dopo la rimozione: " + fruits);
            }
        }

Utilizzo di HashMap


        import java.util.HashMap;
        
        public class HashMapExample {
            public static void main(String[] args) {
                HashMap students = new HashMap<>();
                students.put(1, "Mario Rossi");
                students.put(2, "Luca Bianchi");
                students.put(3, "Giulia Verdi");
        
                System.out.println("Studenti: " + students);
                System.out.println("Studente con ID 2: " + students.get(2));
            }
        }

Utilizzo di HashSet


        import java.util.HashSet;
        
        public class HashSetExample {
            public static void main(String[] args) {
                HashSet cities = new HashSet<>();
                cities.add("Roma");
                cities.add("Milano");
                cities.add("Napoli");
                cities.add("Roma"); // Duplicato, non verrà aggiunto
        
                System.out.println("Città: " + cities);
            }
        }

2.1 ArrayList

Un ArrayList è una lista dinamica che può crescere o ridursi in base alla necessità. A differenza degli array tradizionali, non è necessario specificarne una dimensione fissa al momento della creazione. Questa classe è ideale quando si prevede di aggiungere o rimuovere frequentemente elementi da una lista.

Caratteristiche principali:

  • Permette duplicati.
  • Gli elementi sono ordinati in base all'ordine di inserimento.
  • Offre metodi per l'inserimento, la rimozione e la ricerca di elementi.

Esempio di utilizzo:

import java.util.ArrayList;
        
        public class EsempioArrayList {
            public static void main(String[] args) {
                // Creazione di un ArrayList
                ArrayList nomi = new ArrayList<>();
                
                // Aggiunta di elementi
                nomi.add("Mario");
                nomi.add("Luigi");
                nomi.add("Peach");
                
                // Rimozione di un elemento
                nomi.remove("Luigi"); // Rimuove "Luigi"
                
                // Modifica di un elemento
                nomi.set(0, "Yoshi"); // Sostituisce "Mario" con "Yoshi"
                
                // Iterazione sugli elementi
                for (String nome : nomi) {
                    System.out.println(nome);
                }
            }
        }

2.2 HashMap

Un HashMap è una struttura dati che utilizza una tabella hash per memorizzare coppie chiave-valore. Le chiavi devono essere uniche e possono essere utilizzate per recuperare rapidamente i valori associati. Questa classe è particolarmente utile per gestire dati che richiedono associazioni dirette, come un inventario o una rubrica.

Caratteristiche principali:

  • Le chiavi sono uniche, ma i valori possono essere duplicati.
  • Permette l'inserimento, la rimozione e la ricerca in modo efficiente.
  • Non garantisce un ordine specifico per le chiavi o i valori.

Esempio di utilizzo:

import java.util.HashMap;
        
        public class EsempioHashMap {
            public static void main(String[] args) {
                // Creazione di un HashMap
                HashMap inventario = new HashMap<>();
                
                // Aggiunta di coppie chiave-valore
                inventario.put("Mele", 10);
                inventario.put("Banane", 5);
                inventario.put("Arance", 8);
                
                // Aggiornamento di un valore
                inventario.put("Banane", 12); // Aggiorna il valore associato a "Banane"
                
                // Recupero di un valore
                System.out.println("Banane disponibili: " + inventario.get("Banane"));
                
                // Iterazione su chiavi e valori
                for (String chiave : inventario.keySet()) {
                    System.out.println(chiave + ": " + inventario.get(chiave));
                }
            }
        }

2.3 HashSet

Un HashSet è una struttura dati che memorizza elementi unici. È utile quando si desidera evitare duplicati e non si ha bisogno di mantenere un ordine specifico degli elementi.

Caratteristiche principali:

  • Gli elementi sono unici e i duplicati vengono ignorati.
  • Non garantisce l'ordine degli elementi.
  • Offre un'operazione di ricerca rapida.

Esempio di utilizzo:

import java.util.HashSet;
        
        public class EsempioHashSet {
            public static void main(String[] args) {
                // Creazione di un HashSet
                HashSet colori = new HashSet<>();
                
                // Aggiunta di elementi
                colori.add("Rosso");
                colori.add("Verde");
                colori.add("Blu");
                colori.add("Rosso"); // Ignorato perché duplicato
                
                // Iterazione sugli elementi
                for (String colore : colori) {
                    System.out.println(colore);
                }
                
                // Verifica se un elemento esiste
                if (colori.contains("Verde")) {
                    System.out.println("Il colore Verde è presente.");
                }
            }
        }

Sezione Avanzata

Questa sezione è dedicata agli studenti che desiderano approfondire ulteriormente gli argomenti trattati, esplorando funzionalità avanzate e concetti più complessi legati alle collezioni in Java.

1. Framework Collections e Metodi di Sincronizzazione

Il framework delle collezioni offre metodi per sincronizzare le strutture dati e renderle sicure in ambienti multi-thread. Ad esempio, puoi utilizzare Collections.synchronizedList() per creare una lista sincronizzata:

import java.util.ArrayList;
        import java.util.Collections;
        import java.util.List;
        
        public class SynchronizedExample {
            public static void main(String[] args) {
                List listaNonSicura = new ArrayList<>();
                List listaSicura = Collections.synchronizedList(listaNonSicura);
        
                // Aggiunta di elementi in modo sicuro
                listaSicura.add("Elemento 1");
                listaSicura.add("Elemento 2");
        
                synchronized (listaSicura) {
                    for (String elemento : listaSicura) {
                        System.out.println(elemento);
                    }
                }
            }
        }

La sincronizzazione è fondamentale quando si lavora in contesti concorrenti per evitare condizioni di gara o inconsistenze.

2. Creazione di una Struttura Dati Personalizzata

Gli studenti avanzati possono provare a creare una propria struttura dati utilizzando i generics. Ad esempio, una semplice implementazione di una pila (stack):

import java.util.LinkedList;
        
        public class MyStack {
            private LinkedList elementi = new LinkedList<>();
        
            public void push(T elemento) {
                elementi.addFirst(elemento);
            }
        
            public T pop() {
                return elementi.removeFirst();
            }
        
            public boolean isEmpty() {
                return elementi.isEmpty();
            }
        
            public static void main(String[] args) {
                MyStack stack = new MyStack<>();
                stack.push(10);
                stack.push(20);
                stack.push(30);
        
                while (!stack.isEmpty()) {
                    System.out.println(stack.pop());
                }
            }
        }

Questo esercizio aiuta a comprendere i meccanismi interni delle strutture dati e a migliorare le proprie capacità di programmazione.
