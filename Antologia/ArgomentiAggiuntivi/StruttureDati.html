Esercitazione: Strutture Dati: List e Map

Introduzione alle Strutture Dati

Le strutture dati sono un elemento fondamentale nella programmazione e rappresentano un mezzo per organizzare e gestire le informazioni in modo efficiente. In sostanza, consentono di risolvere problemi complessi mantenendo il codice leggibile, scalabile e performante.

In Java, esistono molteplici tipi di strutture dati, ma in questa lezione ci concentreremo su due delle più importanti e ampiamente utilizzate: List e Map. La comprensione di queste due strutture consente di affrontare una vasta gamma di situazioni reali, dalla gestione di elenchi sequenziali alla creazione di associazioni chiave-valore per un rapido accesso ai dati.

Una List è una collezione ordinata, ideale per rappresentare una sequenza di elementi dove l'ordine e i duplicati sono importanti. Ad esempio, una lista della spesa o una sequenza di attività da completare.

Una Map, d'altra parte, è una collezione non ordinata basata su associazioni chiave-valore, perfetta per rappresentare scenari in cui si desidera un accesso rapido ai dati tramite una chiave univoca. Un esempio comune è un dizionario, in cui una parola (la chiave) è associata alla sua definizione (il valore).

Nel corso di questa lezione, analizzeremo le caratteristiche principali, i vantaggi e gli svantaggi di queste due strutture, corredando il tutto con esempi pratici che ne dimostrano l'utilità.

La struttura List

List è una collezione ordinata che consente elementi duplicati. Alcune implementazioni comuni di List in Java sono:

  • ArrayList
  • LinkedList

Esempio pratico

import java.util.ArrayList;
import java.util.List;

public class ListExample {
    public static void main(String[] args) {
        List<String> fruits = new ArrayList<>();

        // Aggiunta di elementi
        fruits.add("Mela");
        fruits.add("Banana");
        fruits.add("Arancia");

        // Iterazione
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // Accesso per indice
        System.out.println("Frutto al primo indice: " + fruits.get(0));
    }
}

La struttura Map

Map è una collezione che associa chiavi a valori. Le chiavi devono essere uniche, ma i valori possono essere duplicati. Alcune implementazioni comuni di Map in Java sono:

  • HashMap
  • TreeMap
  • LinkedHashMap

Esempio pratico

import java.util.HashMap;
import java.util.Map;

public class MapExample {
    public static void main(String[] args) {
        Map<String, Integer> studentGrades = new HashMap<>();

        // Inserimento di chiavi e valori
        studentGrades.put("Mario", 90);
        studentGrades.put("Luigi", 85);
        studentGrades.put("Anna", 92);

        // Iterazione
        for (Map.Entry<String, Integer> entry : studentGrades.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue());
        }

        // Accesso per chiave
        System.out.println("Voto di Mario: " + studentGrades.get("Mario"));
    }
}

Confronto tra List e Map

List e Map sono due strutture dati fondamentali, ciascuna con caratteristiche e utilizzi specifici. La scelta tra di esse dipende dal tipo di problema che si vuole risolvere:

List è ideale per lavorare con una sequenza ordinata di elementi, come una lista della spesa o una coda di attività. Permette di accedere agli elementi in base all'indice e di avere elementi duplicati. Alcuni dei metodi principali di List includono:

  • add(): per aggiungere un elemento alla lista.
  • get(): per accedere a un elemento in base all'indice.
  • remove(): per rimuovere un elemento specifico.
  • size(): per ottenere il numero totale di elementi nella lista.

Esempio pratico di utilizzo di List:

import java.util.ArrayList;
        import java.util.List;
        
        public class ListExample {
            public static void main(String[] args) {
                List tasks = new ArrayList<>();
        
                // Aggiunta di attività
                tasks.add("Comprare il latte");
                tasks.add("Scrivere codice Java");
                tasks.add("Fare una passeggiata");
        
                // Iterazione e stampa
                for (String task : tasks) {
                    System.out.println(task);
                }
        
                // Accesso tramite indice
                System.out.println("Prima attività: " + tasks.get(0));
            }
        }

Map, invece, è perfetta quando si ha bisogno di associare una chiave univoca a un valore. Per esempio, una rubrica telefonica in cui i nomi (chiavi) sono associati ai numeri di telefono (valori). A differenza di List, non consente duplicati per le chiavi, ma permette valori duplicati. Alcuni dei metodi principali di Map includono:

  • put(): per aggiungere una coppia chiave-valore.
  • get(): per ottenere un valore basato sulla chiave.
  • remove(): per rimuovere una coppia chiave-valore specifica.
  • containsKey(): per verificare l'esistenza di una chiave.

Esempio pratico di utilizzo di Map:

import java.util.HashMap;
        import java.util.Map;
        
        public class MapExample {
            public static void main(String[] args) {
                Map phoneBook = new HashMap<>();
        
                // Aggiunta di coppie chiave-valore
                phoneBook.put("Mario", "333-1234567");
                phoneBook.put("Luigi", "333-7654321");
                phoneBook.put("Anna", "333-9876543");
        
                // Iterazione e stampa
                for (Map.Entry entry : phoneBook.entrySet()) {
                    System.out.println(entry.getKey() + ": " + entry.getValue());
                }
        
                // Accesso tramite chiave
                System.out.println("Numero di Mario: " + phoneBook.get("Mario"));
            }
        }

In conclusione:

  • Usa List quando hai bisogno di una collezione ordinata di elementi in cui l'ordine e i duplicati sono importanti.
  • Usa Map quando devi associare una chiave univoca a un valore per un accesso rapido e diretto.
