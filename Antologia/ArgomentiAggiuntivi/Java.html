Lezione: Introduzione a Java

Cos'è Java?

Java è un linguaggio di programmazione ad alto livello, orientato agli oggetti, creato da Sun Microsystems nel 1995 e attualmente mantenuto da Oracle. Il suo obiettivo principale è stato quello di consentire agli sviluppatori di "scrivere una volta, eseguire ovunque" (Write Once, Run Anywhere - WORA). Questo significa che il codice Java, compilato in bytecode, può essere eseguito su qualsiasi piattaforma dotata di una Java Virtual Machine (JVM) compatibile, garantendo così un elevato livello di portabilità.

Java è progettato per essere:

  • Semplice: Con una sintassi simile a quella di C++, ma priva di caratteristiche complesse come i puntatori espliciti.
  • Orientato agli oggetti: Promuove la modularità e la riusabilità del codice grazie a concetti come classi, oggetti, ereditarietà e polimorfismo.
  • Sicuro: La gestione automatica della memoria, un sistema di tipi rigoroso e il sandboxing nella JVM proteggono da molte vulnerabilità.
  • Portabile: Grazie alla JVM, il bytecode può essere eseguito su una vasta gamma di dispositivi e piattaforme.
  • Scalabile e performante: Adatto sia per applicazioni leggere che per grandi sistemi aziendali grazie all'ottimizzazione continua della JVM.

Java è oggi utilizzato in una vasta gamma di applicazioni, tra cui:

  • Applicazioni desktop: Software per ufficio, editor grafici e strumenti per sviluppatori.
  • Applicazioni mobili: La maggior parte delle app Android è sviluppata in Java o in Kotlin, che si integra facilmente con Java.
  • Sistemi embedded: Dispositivi come lettori Blu-ray, set-top box e sensori IoT.
  • Applicazioni web e enterprise: Grandi sistemi aziendali, grazie a framework come Spring e Java EE.
  • Sviluppo di giochi: Anche se meno comune rispetto ad altri linguaggi, Java è utilizzato per alcuni giochi multipiattaforma.

Con il passare degli anni, Java si è evoluto con aggiornamenti regolari, offrendo nuove funzionalità e miglioramenti alle prestazioni, mantenendo una vasta comunità e un ecosistema ricco di librerie e framework per semplificare lo sviluppo e affrontare le esigenze moderne.

La Nascita di Java

Java è stato sviluppato da Sun Microsystems e lanciato ufficialmente nel 1995. Il progetto iniziale, chiamato "Project Green", fu avviato nei primi anni '90 con l'obiettivo di creare un linguaggio di programmazione versatile destinato a dispositivi elettronici di consumo, come set-top box, televisori interattivi e telecomandi.

Il team di sviluppo era guidato da James Gosling, affettuosamente conosciuto come il "padre di Java". Durante lo sviluppo, il linguaggio venne inizialmente chiamato "Oak", ispirandosi a una maestosa quercia (in inglese "oak") che cresceva fuori dall'ufficio di Gosling. Tuttavia, a causa di conflitti con altri marchi registrati, il nome fu cambiato in "Java", prendendo spunto dalla passione del team per il caffè Java, una popolare varietà di caffè proveniente dall'isola di Giava in Indonesia.

Il successo di Java risiede nella sua capacità di soddisfare alcune esigenze fondamentali degli sviluppatori, come:

  • Portabilità: Grazie alla Java Virtual Machine (JVM), i programmi scritti in Java possono essere eseguiti su qualsiasi piattaforma che supporti la JVM, garantendo così un alto grado di compatibilità cross-platform.
  • Semplicità: Java semplifica la programmazione eliminando complessità come i puntatori espliciti e introducendo la gestione automatica della memoria tramite il garbage collector.
  • Sicurezza: L'architettura del linguaggio e la JVM offrono funzionalità per prevenire errori comuni e vulnerabilità, come buffer overflow e accessi non autorizzati.
  • Orientamento agli oggetti: Java si basa su questo paradigma per favorire lo sviluppo di software modulare e riutilizzabile, riducendo i costi di manutenzione a lungo termine.

Nel corso degli anni, Java è diventato uno dei linguaggi di programmazione più popolari al mondo, utilizzato in una vasta gamma di contesti, dalle applicazioni mobili su Android alle grandi applicazioni enterprise, fino ai sistemi embedded e ai server cloud. La sua comunità attiva e il continuo sviluppo lo mantengono rilevante e potente anche nelle tecnologie moderne.

Cos'è la Java Virtual Machine (JVM)?

La Java Virtual Machine (JVM) è uno degli elementi chiave che rendono Java un linguaggio versatile, portabile e sicuro. La JVM è un ambiente di esecuzione virtuale responsabile dell'esecuzione del bytecode, un formato intermedio generato dal compilatore Java. Questo approccio consente al codice Java di essere indipendente dalla piattaforma, poiché il bytecode può essere eseguito su qualsiasi dispositivo dotato di una JVM compatibile.

Quando un programmatore scrive un'applicazione in Java e la compila, il compilatore Java (javac) non produce direttamente codice macchina comprensibile dal sistema operativo, bensì bytecode, un linguaggio binario ottimizzato per la JVM. La JVM si occupa poi di tradurre ed eseguire questo bytecode nel contesto della piattaforma ospitante.

Come funziona la JVM

La JVM opera in diverse fasi, ciascuna delle quali svolge un ruolo importante per garantire che il codice venga eseguito in modo corretto ed efficiente:

  1. Caricamento del bytecode: La JVM utilizza un componente chiamato class loader per caricare le classi Java dal file .class compilato. Queste classi possono essere caricate da diverse fonti, come file system locali o risorse di rete.
  2. Verifica: Prima dell'esecuzione, la JVM verifica che il bytecode sia sicuro e non contenga istruzioni che potrebbero compromettere la stabilità o la sicurezza dell'ambiente.
  3. Interpretazione ed esecuzione: Il bytecode viene inizialmente interpretato dalla JVM, che traduce ogni istruzione in codice macchina specifico per la piattaforma in tempo reale.
  4. Compilazione Just-In-Time (JIT): Per migliorare le prestazioni, la JVM utilizza il compilatore JIT per convertire porzioni di bytecode frequentemente utilizzate direttamente in codice macchina, riducendo i costi di interpretazione continua. Questo processo accelera significativamente l'esecuzione.

Esempi pratici

Per comprendere meglio il funzionamento della JVM, consideriamo un esempio: 


    // Un semplice programma Java
    public class EsempioJVM {
        public static void main(String[] args) {
            System.out.println("Ciao, JVM!");
        }
    }

1. Il codice sopra viene compilato con il comando javac EsempioJVM.java. Questo genera un file EsempioJVM.class, che contiene il bytecode.

2. Quando eseguiamo il comando java EsempioJVM, la JVM carica il file EsempioJVM.class, verifica il bytecode e lo interpreta.

3. Se il metodo main viene richiamato frequentemente, il compilatore JIT potrebbe ottimizzarlo convertendolo in codice macchina nativo, migliorando le prestazioni.

Funzionalità principali della JVM

  • Gestione della memoria: La JVM gestisce automaticamente la memoria tramite il garbage collector, che libera memoria occupata da oggetti non più utilizzati, evitando perdite di memoria.
  • Ottimizzazioni runtime: La compilazione JIT e altre tecniche consentono di migliorare le prestazioni delle applicazioni.
  • Sicurezza: La JVM isola il bytecode dal sistema operativo e verifica che il codice non contenga istruzioni pericolose o non autorizzate.

Interpretazione vs. Compilazione JIT

Un esempio per chiarire la differenza tra interpretazione e compilazione JIT:

  • Interpretazione: La JVM traduce ed esegue il bytecode istruzione per istruzione. Questo approccio è più lento ma consente alla JVM di iniziare subito l'esecuzione del programma.
  • Compilazione JIT: La JVM analizza il codice e identifica le parti eseguite più frequentemente, compilando queste sezioni in codice nativo per la piattaforma, che può essere eseguito molto più velocemente rispetto al bytecode interpretato.

La combinazione di queste tecniche rende la JVM uno strumento potente per eseguire applicazioni Java in modo efficiente su diverse piattaforme.

Esempio Pratico

Un semplice programma Java per mostrare il classico "Hello, World!". Questo esempio è il punto di partenza per qualsiasi principiante che desideri apprendere Java, poiché illustra la struttura base di un programma Java: 


    // Definizione della classe pubblica
    public class HelloWorld {
        // Metodo principale, punto di ingresso del programma
        public static void main(String[] args) {
            // Stampa del messaggio "Hello, World!" sulla console
            System.out.println("Hello, World!");
        }
    }

Spiegazione del codice

  • public class HelloWorld: Ogni programma Java deve avere almeno una classe. Qui, la classe si chiama HelloWorld. Il nome del file deve corrispondere al nome della classe pubblica (in questo caso HelloWorld.java).
  • public static void main(String[] args): Questo è il metodo principale, cioè il punto di ingresso del programma. Quando esegui un programma Java, la JVM cerca e avvia questo metodo.
  • System.out.println("Hello, World!"); Questo comando stampa il messaggio "Hello, World!" sulla console. La funzione println aggiunge automaticamente un ritorno a capo dopo la stringa stampata.

Come eseguire il programma

Per compilare ed eseguire il programma, segui questi passaggi:

  1. Salva il codice in un file chiamato HelloWorld.java.
  2. Compila il file con il comando: 
    javac HelloWorld.java
    Questo comando genera un file HelloWorld.class contenente il bytecode.
  3. Esegui il programma con il comando: 
    java HelloWorld
    Questo comando avvierà la JVM, che caricherà ed eseguirà il file HelloWorld.class, producendo il seguente output sulla console: 

    Hello, World!

Concetti chiave mostrati in questo esempio

  • Compilazione: Il comando javac converte il codice sorgente Java in bytecode eseguibile dalla JVM.
  • Esecuzione: Il comando java esegue il bytecode sulla JVM, rendendo il programma indipendente dalla piattaforma.
  • Struttura del programma: La struttura base di un programma Java include una classe, un metodo principale e istruzioni eseguibili.

Questo semplice esempio pone le basi per comprendere i concetti fondamentali della programmazione Java, come classi, metodi e l'interazione con la console.
