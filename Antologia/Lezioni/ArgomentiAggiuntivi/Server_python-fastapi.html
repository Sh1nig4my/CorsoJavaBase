Progetto: Creazione di un Server Python con FastAPI

1. Introduzione a Python e FastAPI

**FastAPI** è un framework moderno, leggero e altamente performante per la creazione di **API RESTful** in Python. Basato sui **Python type hints**, è progettato per offrire **validazione automatica dei dati**, **documentazione interattiva**, e **massime prestazioni** grazie all'uso di **Starlette** e **Pydantic**. FastAPI è particolarmente indicato per applicazioni che richiedono efficienza, scalabilità e semplicità nello sviluppo.

🚀 Perché scegliere FastAPI?

Rispetto ad altri framework come Flask o Django REST Framework, FastAPI si distingue per **prestazioni eccezionali** e un approccio basato su **tipizzazione statica**, che migliora lo sviluppo e la qualità del codice.

  • ⚡ Alte prestazioni: Grazie all'uso di **Starlette** (per il server) e **Pydantic** (per la validazione dei dati), FastAPI è fino a 3 volte più veloce di Flask e comparabile a Node.js.
  • 🛠 Facilità di sviluppo: L'autocompletamento avanzato e il supporto nativo per **type hints** riducono gli errori e migliorano la leggibilità del codice.
  • 📑 Documentazione automatica: Ogni API creata con FastAPI genera automaticamente una documentazione interattiva utilizzando **Swagger UI** e **ReDoc**, accessibile direttamente via browser.
  • 🛡 Sicurezza integrata: Supporto nativo per **OAuth2**, autenticazione JWT e middleware di sicurezza.
  • 🔌 Asincronia nativa: FastAPI supporta **async/await**, permettendo di gestire operazioni I/O in modo efficiente e migliorando le prestazioni in applicazioni ad alto traffico.

📌 Quando usare FastAPI?

FastAPI è ideale per la creazione di:

  • 📡 **API REST ad alte prestazioni** per microservizi.
  • 🤖 **Sistemi di machine learning** con modelli di AI serviti via API.
  • 🔄 **Backend per applicazioni web** e mobile con gestione di dati in tempo reale.
  • 💾 **Servizi basati su database** grazie all'integrazione con SQLAlchemy e MongoDB.

🎯 Conclusione

Grazie alla sua **semplicità, velocità e sicurezza**, FastAPI è uno dei framework più consigliati per la realizzazione di **API moderne e scalabili**. Nei prossimi passi vedremo come installarlo e creare la nostra prima API con FastAPI. 🚀

2. Installazione di Python e Configurazione dell'Ambiente

2.1 Installazione di Python

Per sviluppare con **FastAPI**, è necessario installare **Python 3.7+**. Puoi scaricare l'ultima versione dal sito ufficiale: Python Download.

📌 Passaggi per installare Python:

  1. Scarica l'installer per il tuo sistema operativo (Windows, macOS o Linux).
  2. Su **Windows**, durante l'installazione, seleziona l'opzione "Add Python to PATH" per facilitarne l'uso da terminale.
  3. Completa l'installazione e verifica che Python sia stato installato correttamente eseguendo il seguente comando nel terminale:

# Verifica della versione di Python
python --version

# Verifica della versione di pip (gestore pacchetti di Python)
pip --version

Se i comandi restituiscono i numeri di versione di Python e pip, significa che l'installazione è avvenuta con successo. 🎉

2.2 Creazione di un Progetto Python per FastAPI

Per organizzare al meglio il nostro progetto FastAPI, creiamo un ambiente virtuale. Questo ci permetterà di gestire le dipendenze senza interferire con altri progetti Python sul sistema.

📌 Creazione della cartella del progetto:


mkdir my-fastapi-server
cd my-fastapi-server

📌 Creazione di un ambiente virtuale:

Un **ambiente virtuale** isola le dipendenze del progetto. Creiamolo eseguendo: 


# Creazione dell'ambiente virtuale
python -m venv venv

📌 Attivazione dell'ambiente virtuale:

A seconda del sistema operativo, usa uno dei seguenti comandi per attivare l'ambiente virtuale:

  • Linux/macOS:
    • source venv/bin/activate
  • Windows (cmd):
    • venv\Scripts\activate
  • Windows (PowerShell):
    • venv\Scripts\Activate.ps1

Quando l'ambiente è attivato, vedrai il prefisso **(venv)** nel terminale. Questo indica che i pacchetti installati verranno usati solo all'interno di questo progetto.

2.3 Installazione di FastAPI e Uvicorn

**FastAPI** è il framework che useremo per creare l'API, mentre **Uvicorn** è il server ASGI che ci permetterà di eseguire l'applicazione. Installiamoli utilizzando **pip**: 


pip install fastapi uvicorn

📌 Verifica dell'installazione:

Dopo l'installazione, possiamo verificare che FastAPI sia stato installato correttamente eseguendo: 


pip list | grep fastapi

Se vedi **fastapi** e **uvicorn** nella lista, significa che l'installazione è andata a buon fine! 🎉

📌 Conclusione

Ora abbiamo un ambiente Python configurato con un **ambiente virtuale isolato** e **FastAPI installato**. Nel prossimo passo, creeremo la nostra prima API con FastAPI e la eseguiremo con Uvicorn. 🚀

3. Creazione di un Server con FastAPI

3.1 Creazione del file main.py

Ora che abbiamo installato **FastAPI** e **Uvicorn**, possiamo creare il nostro primo server API. FastAPI è progettato per essere semplice e performante, permettendo di definire API con poche righe di codice.

📌 Passaggi per creare il server:

  1. Apri il terminale e assicurati di essere nella cartella del progetto cd my-fastapi-server.
  2. Crea un nuovo file Python chiamato **main.py**:
touch main.py

Apri il file con il tuo editor di testo preferito (VS Code, PyCharm, Sublime, Nano) e aggiungi il seguente codice:

3.2 Scrittura del codice per il server

Il codice seguente crea un server **FastAPI** con un **endpoint di base** che restituisce un messaggio JSON di benvenuto. 


from fastapi import FastAPI

# Creazione dell'applicazione FastAPI
app = FastAPI()

# Definizione di una route GET per la home
@app.get("/")
def read_root():
    return {"message": "Benvenuto nel server FastAPI!"}

# Avvio del server con Uvicorn se lo script viene eseguito direttamente
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="127.0.0.1", port=8000)

📌 Spiegazione del codice:

  • from fastapi import FastAPI: Importa FastAPI.
  • app = FastAPI(): Crea un'istanza dell'applicazione.
  • @app.get("/"): Definisce una route **GET** che risponde con un messaggio JSON.
  • def read_root(): La funzione che restituisce il messaggio di benvenuto.
  • if __name__ == "__main__": Permette di avviare il server direttamente da **Python**, utile per il debug.
  • uvicorn.run(app, host="127.0.0.1", port=8000): Avvia il server su **localhost:8000**.

3.3 Avvio del Server con Uvicorn

Per eseguire il server, utilizzeremo **Uvicorn**, un server ASGI ottimizzato per applicazioni FastAPI. Esegui il seguente comando nel terminale: 

uvicorn main:app --reload

📌 Cosa fa il comando?

  • uvicorn: Avvia il server ASGI.
  • main:app: Specifica il file main.py e l'istanza **app** da eseguire.
  • --reload: Abilita il **riavvio automatico** quando il codice viene modificato (utile in fase di sviluppo).

Se tutto è configurato correttamente, il terminale mostrerà un output simile a questo: 


INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.

3.4 Test del Server

Apri il browser e vai su **http://127.0.0.1:8000**. Dovresti vedere una risposta JSON come questa: 


{
    "message": "Benvenuto nel server FastAPI!"
}

3.5 Documentazione Automatica

Uno dei vantaggi di **FastAPI** è la generazione automatica della documentazione API. Puoi accedervi direttamente aprendo il browser su:

  • 📄 **Swagger UI**: http://127.0.0.1:8000/docs
  • 📄 **ReDoc**: http://127.0.0.1:8000/redoc

Questi strumenti permettono di **esplorare e testare le API** in modo interattivo direttamente dal browser. 🚀

📌 Conclusione

Ora hai un server **FastAPI** funzionante con **documentazione automatica** e supporto per richieste **RESTful**. 🎉 Nei prossimi passi, vedremo come aggiungere **parametri alle route**, gestire **richieste POST** e collegare un **database** al nostro server! 🔥

4. Creazione delle API di Login e Logout

In questa sezione, implementeremo due endpoint API per gestire l'autenticazione di un utente utilizzando **FastAPI**:

  • 🔑 Un'API **POST** per il **login**, che verifica le credenziali dell'utente e restituisce una risposta JSON.
  • 🚪 Un'API **POST** per il **logout**, che semplicemente conferma l'uscita dell'utente.

4.1 Creazione dell'API di Login

L'endpoint di login riceve un **username** e una **password** dal corpo della richiesta e verifica se le credenziali sono corrette. Se l'autenticazione ha successo, restituisce una risposta positiva; altrimenti, restituisce un errore HTTP **401 Unauthorized**.

📌 Implementazione dell'API Login


from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI()

# Modello dati per la richiesta di login
class LoginRequest(BaseModel):
    username: str
    password: str

@app.post("/login")
def login(request: LoginRequest):
    # Simulazione della verifica delle credenziali
    if request.username == "admin" and request.password == "password":
        return {"success": True, "message": "Login riuscito!"}

    # Se le credenziali sono errate, restituisce un errore HTTP 401
    raise HTTPException(status_code=401, detail="Credenziali errate")

📌 Spiegazione del codice:

  • from fastapi import FastAPI, HTTPException: Importa FastAPI e la gestione delle eccezioni.
  • from pydantic import BaseModel: Pydantic permette di definire e validare i dati di input.
  • class LoginRequest(BaseModel): Definisce la struttura della richiesta di login.
  • @app.post("/login"): Definisce un endpoint **POST** che gestisce la richiesta di login.
  • if request.username == "admin" and request.password == "password": Simula un controllo sulle credenziali dell'utente.
  • raise HTTPException(status_code=401, detail="Credenziali errate"): Se le credenziali sono errate, viene restituito un errore HTTP 401.

🛠 Test dell'API con Postman o cURL

Per testare il login, puoi usare **Postman** o **cURL** per inviare una richiesta **POST** con un corpo JSON come questo: 


{
    "username": "admin",
    "password": "password"
}

Se le credenziali sono corrette, la risposta sarà: 


{
    "success": true,
    "message": "Login riuscito!"
}

Se le credenziali sono errate, la risposta sarà: 


{
    "detail": "Credenziali errate"
}

4.2 Creazione dell'API di Logout

L'endpoint di logout restituisce semplicemente un messaggio di conferma. In un'applicazione reale, potrebbe anche invalidare un **token di sessione** o eliminare un **cookie di autenticazione**.

📌 Implementazione dell'API Logout


@app.post("/logout")
def logout():
    return {"success": True, "message": "Logout effettuato con successo!"}

📌 Spiegazione del codice:

  • @app.post("/logout"): Definisce un endpoint **POST** per gestire il logout.
  • return {"success": True, "message": "Logout effettuato con successo!"}: Restituisce una conferma del logout.

🛠 Test dell'API Logout

Per testare il logout, invia una richiesta **POST** a **http://127.0.0.1:8000/logout**. La risposta attesa sarà: 


{
    "success": true,
    "message": "Logout effettuato con successo!"
}
