Concetti base e di Introduzione a Java

1. Introduzione

Java è un linguaggio di programmazione di alto livello, orientato agli oggetti, sviluppato da Sun Microsystems nel 1995 e attualmente mantenuto da Oracle. È stato progettato con l'obiettivo di essere portabile, sicuro e robusto, consentendo agli sviluppatori di scrivere codice una sola volta ed eseguirlo su qualsiasi piattaforma dotata di una Java Virtual Machine (JVM). Questo principio è noto come il paradigma "Write Once, Run Anywhere" (WORA).

Java è ampiamente utilizzato nello sviluppo di applicazioni desktop, mobile (Android), enterprise e sistemi embedded. La sua sintassi è ispirata a C++, ma con una gestione della memoria più sicura grazie al garbage collector, che libera automaticamente la memoria inutilizzata.

Uno degli elementi distintivi di Java è la sua architettura basata su tre componenti fondamentali:

  • JDK (Java Development Kit): Fornisce gli strumenti necessari per sviluppare applicazioni Java, tra cui il compilatore (javac), la documentazione e le librerie.
  • JRE (Java Runtime Environment): Include tutto il necessario per eseguire programmi Java, tra cui la JVM e le librerie di base.
  • JVM (Java Virtual Machine): Il cuore del linguaggio, che interpreta ed esegue il bytecode compilato da Java su qualsiasi sistema operativo.

2. Concetti Chiave

Per comprendere Java a fondo, è essenziale conoscere alcuni concetti chiave che ne caratterizzano il funzionamento e la filosofia di programmazione.

  • Paradigma Orientato agli Oggetti (OOP): Java è un linguaggio completamente orientato agli oggetti, il che significa che tutto in Java è basato su classi e oggetti. L'OOP facilita il riutilizzo del codice, la modularità e la manutenzione del software. I quattro pilastri fondamentali dell'OOP in Java sono:
    • Incapsulamento: Protegge i dati racchiudendoli all'interno di classi e fornendo metodi pubblici per l'accesso controllato.
    • Ereditarietà: Permette di definire nuove classi basate su classi esistenti, riducendo la duplicazione del codice.
    • Polimorfismo: Consente alle classi di essere utilizzate attraverso una comune interfaccia, migliorando la flessibilità e la scalabilità del codice.
    • Astrazione: Permette di nascondere i dettagli di implementazione e di esporre solo le funzionalità rilevanti.
  • JVM (Java Virtual Machine): La JVM è un ambiente di esecuzione che permette ai programmi Java di essere eseguiti su qualsiasi sistema operativo senza modifiche. Converte il bytecode in istruzioni macchina specifiche per il sistema in cui viene eseguito, garantendo portabilità e sicurezza.
  • Bytecode: Il codice sorgente Java non viene compilato direttamente in codice macchina, ma in un formato intermedio chiamato bytecode. Questo bytecode viene poi interpretato dalla JVM, il che lo rende indipendente dal sistema operativo.
  • Gestione della Memoria: Java utilizza un Garbage Collector (GC) per gestire automaticamente la memoria, eliminando oggetti non più utilizzati e riducendo il rischio di memory leaks.
  • Exception Handling: Java offre un robusto sistema di gestione delle eccezioni che consente di scrivere codice più sicuro e meno soggetto a errori imprevedibili. Le eccezioni vengono gestite tramite blocchi try, catch e finally.
  • Multithreading: Java supporta il multithreading, consentendo l'esecuzione concorrente di più thread per migliorare le prestazioni delle applicazioni.
  • Framework e Librerie: Java dispone di una vasta gamma di librerie standard e framework per lo sviluppo di applicazioni, tra cui:
    • Java Standard Edition (SE): Fornisce le API di base.
    • Java Enterprise Edition (EE): Offerte avanzate per lo sviluppo di applicazioni aziendali.
    • Spring: Un framework popolare per lo sviluppo di applicazioni enterprise.
