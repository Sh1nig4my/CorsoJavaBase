Lezione 11: Correzione, Discussione Errori e Refactoring del Codice

1. Introduzione

Durante lo sviluppo di software, è inevitabile incontrare errori e problemi nel codice. Una parte essenziale del lavoro di un programmatore è la capacità di individuare, comprendere e correggere tali errori in modo efficace. In questa lezione analizzeremo gli errori comuni, discuteremo strategie per la loro correzione e introdurremo le tecniche di refactoring per migliorare la qualità del codice.

2. Tipologie di Errori in Java

  • Errori di sintassi: Sono causati da errori nella scrittura del codice e impediscono la compilazione.
  • Errori di runtime: Si verificano durante l'esecuzione del programma, ad esempio divisioni per zero o NullPointerException.
  • Errori logici: Il programma viene eseguito senza errori apparenti, ma il comportamento non è quello atteso.

Esempio di Errore di Runtime

public class ErroreRuntime {
    public static void main(String[] args) {
        String testo = null;
        System.out.println(testo.length()); // Genera NullPointerException
    }
}

3. Tecniche di Debugging

Per individuare e correggere gli errori è utile seguire alcuni passi fondamentali:

  • Utilizzare i messaggi di errore del compilatore per individuare problemi sintattici.
  • Inserire stampe di debug con System.out.println per tracciare il flusso del programma.
  • Utilizzare un debugger in un IDE come IntelliJ o Eclipse per analizzare lo stato delle variabili.

4. Introduzione al Refactoring

Il refactoring è il processo di miglioramento del codice senza modificarne il comportamento esterno. È utile per aumentare la leggibilità, ridurre la complessità e migliorare la manutenibilità.

Principi del refactoring:

  • Eliminare la ridondanza
  • Rinominare variabili e metodi per aumentarne la comprensibilità
  • Estrarre metodi per suddividere il codice in blocchi più gestibili

Esempio di Refactoring

public class Refactoring {
    public static void main(String[] args) {
        int somma = calcolaSomma(10, 20);
        System.out.println("La somma è: " + somma);
    }

    public static int calcolaSomma(int a, int b) {
        return a + b;
    }
}

Esercizio Pratico

Refattorizza il seguente codice per renderlo più leggibile e manutenibile:

public class EsercizioRefactoring {
    public static void main(String[] args) {
        int x = 5;
        int y = 10;
        System.out.println("Risultato: " + (x * y));
