Programma Corso di Programmazione Backend

Benvenuti al corso di formazione in Programmazione Backend, un viaggio formativo progettato per fornire competenze solide e pratiche nello sviluppo di applicazioni lato server. Questo corso è dedicato a chi desidera intraprendere una carriera come sviluppatore back-end o approfondire le proprie conoscenze in questo ambito, partendo da una base di utilizzo del computer.

Il backend è il cuore di ogni applicazione moderna. È qui che i dati vengono gestiti, le logiche applicative prendono vita e le richieste degli utenti trovano risposta. Durante il corso esploreremo i fondamenti della programmazione backend, i linguaggi e i framework principali, i database relazionali e non relazionali, e le pratiche di testing, culminando nella realizzazione di un progetto completo in Java Spring.


Settimana 1: Introduzione alla Programmazione e Configurazione degli Strumenti

  • Giorno 1:
    - Introduzione alla programmazione: concetti fondamentali, differenza tra frontend e backend.
    - Strumenti di sviluppo: installazione di Java, Maven, Git, e configurazione di un IDE (IntelliJ/Eclipse).
  • Giorno 2:
    - Git: comandi base (clone, add, commit, push, pull).
    - Repository GitHub: creazione di un semplice repository condiviso.
    - Risoluzione problemi: introduzione a pseudocodice e approccio alla risoluzione di problemi.
  • Giorno 3:
    - Fondamenti Java: variabili, tipi di dati, operatori, strutture di controllo.
    - Primo programma Java: input/output da console.
  • Giorno 4:
    - Programmazione orientata agli oggetti: classi, oggetti, ereditarietà, polimorfismo.
    - Esempi OOP: applicazione pratica in Java per modellare un problema reale.
  • Giorno 5 (Esercitazione):
    - Progetto OOP: realizzazione di un programma con gestione di classi e metodi.
    - Versionamento con Git: utilizzo per il progetto sviluppato.

Settimana 2: Tutte le differenze introdotte con Java 8

  • Giorno 6:
    - Lambda Expressions: sintassi e utilizzo, confronto con classi anonime.
    - Functional Interfaces: ruolo e applicazioni pratiche.
    - Default Methods: aggiunta di metodi predefiniti nelle interfacce.
  • Giorno 7:
    - Stream API: operazioni intermedie e terminali, differenze rispetto alle collezioni tradizionali.
    - Optional: utilizzo per evitare null e NullPointerException.
    - Metodi statici nelle interfacce: novità e differenze con approcci precedenti.
  • Giorno 8:
    - Date and Time API: introduzione di java.time, confronto con java.util.Date e Calendar.
    - Parallel Streams: gestione del parallelismo e confronto con tecniche tradizionali.
    - Collector API: creazione di collezioni personalizzate e aggregazione di dati.
  • Giorno 9:
    - Nuovi metodi utility: aggiunte significative in Arrays, Collections e Objects.
    - Metodi di ordinamento: miglioramenti nei metodi di ordinamento per collezioni.
    - Riflessione sulle novità: come Java 8 ha cambiato il paradigma di programmazione.
  • Giorno 10 (Esercitazione):
    - Progetto Java: utilizzo di Stream, Optional e Date API per analisi dati e gestione temporale.
    - Refactoring: modernizzazione di codice legacy utilizzando le novità di Java 8.

Settimana 3: Java + Database

  • Giorno 11:
    - Database Relazionali: introduzione a SQL, tabelle, colonne, righe e query di base.
    - Configurazione: installazione di un database SQL-like (MySQL/PostgreSQL).
  • Giorno 12:
    - Database NoSQL: differenze rispetto ai database relazionali.
    - MongoDB: installazione, configurazione e operazioni CRUD di base.
  • Giorno 13:
    - Collegamento Java-Database: utilizzo di JDBC per SQL e MongoDB Driver.
    - CRUD in Java: query SQL e connessione a MongoDB.
    - Accenno a JPA: utilizzo con SQL e MongoDB.
  • Giorno 14:
    - ORM: introduzione a Hibernate (SQL) e JPA per MongoDB.
    - Configurazione: esempi pratici per entrambi i database.
  • Giorno 15 (Esame Intermedio):
    - Esame Intermedio: verifica delle competenze su OOP, novità Java 8/11 e collegamento ai database SQL e NoSQL.
    - Struttura: esercizio pratico e discussione teorica.

Settimana 4: Java + Spring

  • Giorno 16:
    - Spring Boot: introduzione e creazione di un progetto base.
    - Dependency Injection: concetti fondamentali e configurazione.
  • Giorno 17:
    - Controller e Service: gestione richieste HTTP e logica applicativa.
    - Layer Repository: connessione a MongoDB con Spring Data.
  • Giorno 18:
    - API REST: creazione di endpoints CRUD con Spring Boot.
    - Testing: utilizzo di Postman per testare le API.
  • Giorno 19:
    - Architettura backend: monolitiche vs microservizi.
    - Comunicazione tra microservizi: esempio pratico in Spring.
  • Giorno 20 (Esercitazione):
    - Server REST completo: gestione CRUD su MongoDB con Spring Boot.
    - Testing avanzato: utilizzo di Postman o Swagger.

Settimana 5: Panoramica di Altri Linguaggi e Testing

  • Giorno 21:
    - Python e FastAPI: introduzione e creazione di una semplice API REST.
  • Giorno 22:
    - Node.js ed Express: introduzione e creazione di una semplice API REST.
  • Giorno 23:
    - Confronto framework: Spring vs FastAPI vs Express.
    - Scelta del framework: criteri basati sul contesto applicativo.
  • Giorno 24:
    - Testing in Java: test unitari con JUnit.
    - API REST: accenni a test automatici.
  • Giorno 25 (Esercitazione):
    - Test Unitari: Revisione e implementazione di test unitari e integrazione per le API sviluppate durante il corso.
    - Esempio API REST: Progettazione di un esempio pratico di API REST usando Spring Boot con focus sui concetti appresi.

Settimana 6: Progetto Finale

  • Giorno 26:
    - Progetto finale: brainstorming, definizione entità e architettura.
    - Struttura base: creazione con Spring Boot.
  • Giorno 27:
    - CRUD: implementazione delle funzionalità principali.
    - Database: connessione a MongoDB per la gestione dei dati.
  • Giorno 28:
    - Ottimizzazione: aggiunta di sicurezza, logging e test.
    - Debug: risoluzione dei problemi e miglioramenti finali.
  • Giorno 29:
    - Documentazione: specifiche tecniche, diagrammi e API docs.
    - Preparazione: organizzazione per la presentazione finale.
  • Giorno 30 (Esame e Valutazione):
    - Presentazione progetti: discussione individuale dei progetti sviluppati.
    - Esame orale: analisi critica, pro e contro delle soluzioni proposte.
