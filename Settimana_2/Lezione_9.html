Lezione 9: Novità in Java 8 - Metodi Utility e Paradigmi di Programmazione

La nona lezione si concentra sulle nuove funzionalità introdotte in Java 8 nei metodi utility delle principali classi e sul cambiamento paradigmatico nella programmazione Java. Analizzeremo gli aggiornamenti a Arrays, Collections, e Objects, e discuteremo come questi miglioramenti abbiano influenzato il modo di scrivere codice Java. Con l'adozione di Java 8, gli sviluppatori hanno avuto accesso a strumenti più potenti per scrivere codice conciso, performante e mantenibile, sfruttando anche le capacità dei sistemi multicore grazie a un miglior supporto al parallelismo.

Nuovi Metodi Utility

Java 8 ha introdotto nuovi metodi statici in diverse classi utility, facilitando operazioni comuni e riducendo il codice boilerplate. Approfondiamo i principali miglioramenti:

Arrays

La classe Arrays è stata arricchita con nuovi metodi per la gestione efficiente degli array. Tra questi spicca parallelSort(), che permette di ordinare gli elementi di un array sfruttando il parallelismo. Questo è particolarmente utile per grandi dataset, dove l'elaborazione parallela può ridurre significativamente i tempi di esecuzione.

Esempio 1: Ordinamento parallelo

int[] numeri = {5, 3, 8, 1, 9, 2};

// Ordinamento parallelo
Arrays.parallelSort(numeri);

// Stampa dell'array ordinato
for (int numero : numeri) {
    System.out.println(numero);
}

Esempio 2: Ordinamento personalizzato

int[] numeri = {12, 7, -5, 3};

// Ordinamento con Comparator
Arrays.sort(numeri, (a, b) -> Integer.compare(b, a));

// Stampa dell'array ordinato
for (int numero : numeri) {
    System.out.println(numero);
}

Collections

Le collezioni in Java hanno ricevuto nuovi metodi statici per supportare operazioni comuni. Metodi come emptyList(), emptyMap() e emptySet() creano collezioni immutabili vuote, utili per scenari in cui vogliamo evitare modifiche accidentali.

Esempio: Collezioni vuote

List listaVuota = Collections.emptyList();
System.out.println("Lista vuota: " + listaVuota);

Set setVuoto = Collections.emptySet();
System.out.println("Set vuoto: " + setVuoto);

Objects

La classe Objects introduce metodi per migliorare la gestione degli oggetti. Tra questi troviamo requireNonNull(), che consente di validare i riferimenti nulli, e compare(), utile per confrontare due oggetti con un Comparator fornito.

Esempio 1: Validazione di oggetti

String nome = "Mario";
Objects.requireNonNull(nome, "Il nome non può essere null");
System.out.println("Nome validato: " + nome);

Esempio 2: Confronto di oggetti

String a = "Ciao";
String b = "Mondo";

int risultato = Objects.compare(a, b, String::compareTo);
System.out.println("Risultato del confronto: " + risultato);

Metodi di Ordinamento

I metodi di ordinamento in Java 8 supportano l'uso di espressioni lambda per definire criteri di confronto in modo conciso. Questo permette di ordinare collezioni in modo personalizzato senza la necessità di creare classi Comparator separate.

Esempio: Ordinamento con lambda

List nomi = Arrays.asList("Anna", "Luca", "Marco");

// Ordinamento alfabetico inverso
nomi.sort((a, b) -> b.compareTo(a));

// Stampa della lista ordinata
System.out.println(nomi);

Esempio avanzato: Ordinamento di oggetti

class Persona {
    String nome;
    int eta;

    Persona(String nome, int eta) {
        this.nome = nome;
        this.eta = eta;
    }

    @Override
    public String toString() {
        return nome + " (" + eta + " anni)";
    }
}

List persone = Arrays.asList(
    new Persona("Anna", 30),
    new Persona("Luca", 25),
    new Persona("Marco", 35)
);

// Ordinamento per età
persone.sort((p1, p2) -> Integer.compare(p1.eta, p2.eta));

// Stampa delle persone ordinate
persone.forEach(System.out::println);

Riflessione sulle Novità

Java 8 rappresenta un cambio di paradigma verso uno stile di programmazione più funzionale. Le interfacce funzionali, le espressioni lambda e le nuove API incoraggiano uno sviluppo più conciso e intuitivo. Inoltre, l'introduzione di metodi statici e collezioni immutabili favorisce la scrittura di codice più sicuro e privo di errori legati alla concorrenza.

Con questi miglioramenti, gli sviluppatori possono scrivere codice che si adatta meglio alle esigenze moderne, dove il parallelismo e la modularità giocano un ruolo cruciale.
